《多屏幕适配系列(一)》Viewport

发表于 2017-02-27

desktop browsers

设备像素 and CSS像素

设备像素 对应 screen.width/height

  • 100% 缩放级别，CSS像素和设备像素完全重叠。
  • 缩小(zoom out)，多个CSS像素交叠于一个设备像素。
  • 放大(zoom in)，一个CSS像素占据多个设备像素。

Screen size

用户屏幕的尺寸：screen.width/height

Meaning Total size of the user’s screen.
Measured in Device pixels
Browser errors IE8 measures it in CSS pixels, in both IE7 and IE8 mode.

Window size

浏览器窗口的尺寸：window.innerWidth/innerHeight

Meaning Total size of the browser window, including scrollbars.
Measured in CSS pixels
Browser errors Not supported by IE. Opera measures it in device pixels.

Scrolling offset

相对于窗口的偏移距离：window.pageX/YOffset

Meaning Scrolling offset of the page.
Measured in CSS pixels
Browser errors None

Viewport

Viewport 的尺寸：document. documentElement. clientWidth/Height

Meaning Viewport dimensions
Measured in CSS pixels
Browser errors None

与 window size 相比，Viewport 不包含 scrollbar。

element 占据 Viewport 100% 的宽度。在 Desktop，Viewport 拥有浏览器的宽度和高度，不同于 Desktop，移动端有些复杂。

element

页面元素的大小：document. documentElement.offsetWidth/Height

Meaning Dimensions of the element (and thus of the page).
Measured in CSS pixels
Browser errors IE measures the viewport, and not the element.

Screen -> Browser -> Viewport -> Element ->

Event coordinates

  • pageX/Y 相对于 ，基于CSS像素
  • clientX/Y 相对于 viewport，基于CSS像素
  • screenX/Y 相对于 screen，基于设备像素

media queries

针对不同页面大小定义特殊的 CSS 规则。

与媒体查询相关：

  • width/height documentElement .clientWidth/Height, Viewport, CSS 像素;
  • device-width/device-height screen.width/height, Screen, 设备像素;

Mobile

Visual viewport 是当前屏幕显示页面的一部分。
Layout viewport 比 Visual viewport 广泛，最初 元素获取的是 Layout viewport 的大小，CSS 明显的比手机屏幕广泛。

Symbian WebKit 尝试保持 Layout viewport 等于 Visual viewport，如果由于绝对宽度，页面不符合视觉窗口允许浏览器延伸布局视口最多850 px。
Sanmsung 定义 Layout viewport 的大小为页面中元素最大的值。
BlackBerry 的Layout viewport 等于 Visual viewport。

Layout viewport

浏览器选择布局视口的尺寸,使其处于完全缩放模式来完全覆盖屏幕(并因此等于 Visual viewport)。在放大模式，Layout viewport 的尺寸仍然跟页面元素的尺寸保持一致。

document. documentElement. clientWidth/Height

Meaning Layout viewport dimensions
Measured in CSS pixels
Full support Opera, iPhone, Android, Symbian, Bolt, MicroB, Skyfire, Obigo
Problems

  1. Visual viewport dimensions in Iris
  2. Samsung WebKit reports the correct values when a tag is applied to the page; the dimensions of the element otherwise.
  3. Screen dimensions in device pixels in Firefox
  4. IE returns 1024x768. However, it stores the information in document.body.clientWidth/Height. This is consistent with IE6 desktop.
  5. NetFront’s values are only correct at 100% zoom.
  6. Symbian WebKit 1 (older S60v3 devices) does not support these properties.
    Not supported BlackBerry

Visual viewport

视图可见区域

window.innerWidth/Height

Meaning Visual viewport dimensions
Measured in CSS pixels
Full support iPhone, Symbian, BlackBerry
Problems

  1. Opera and Firefox return the screen width in device pixels.
  2. Android, Bolt, MicroB, and NetFront return the layout viewport dimensions in CSS pixels.
  3. Not supported
  4. IE, but it gives the visual viewport dimension in document. documentElement. offsetWidth/Height.
  5. Samsung WebKit reports either the dimensions of the layout viewport or of the , depending on whether a tag has been applied to the page or not.
    Gibberish Iris, Skyfire, Obigo

The screen

screen.width/height

Meaning Screen size
Measured in Device pixels
Full support Opera Mini, Android, Symbian, Iris, Firefox, MicroB, IE, BlackBerry
Problems

  1. Opera Mobile on Windows Mobile only gives the landscape size. Opera Mobile on S60 gets it right.
  2. Samsung WebKit reports either the dimensions of the layout viewport or of the , depending on whether a tag has been applied to the page or not.
  3. iPhone and Obigo only give portrait sizes.
  4. NetFront only gives landscape sizes.
    Gibberish Bolt, Skyfire

Scrolling offset

window. pageX / YOffset

Meaning Scrolling offset; which is the same as the visual viewport’s offset relative to the layout viewport.
Measured in CSS pixels
Full support iPhone, Android, Symbian, Iris, MicroB, Skyfire, Obigo.
Problems

  1. Opera, Bolt, Firefox, and NetFront always return 0.
  2. Samsung WebKit reports correct values only if a is applied to the page.
    Not supported IE, BlackBerry. IE stores the values in document. documentElement. scrollLeft / Top

element

document. documentElement. offsetWidth / Height

Meaning Total size of the element.
Measured in CSS pixels
Full support Opera, iPhone, Android, Symbian, Samsung, Iris, Bolt, Firefox, MicroB, Skyfire, BlackBerry, Obigo.
Problems

  1. NetFront’s values are only correct at 100% zoom.
  2. IE uses this propery pair to store the dimensions of the visual viewport. In IE, see document. body. clientWidth/Height for the correct values.

Event coordinates

pageX/Y 相对于 , 基于CSS像素
clientX/Y 相对于 Visual viewport, 基于CSS像素
screenX/Y 相对于 screen, 基于设备像素

Media queries

Meaning Measure element width (CSS pixels) or device width (device pixels).
Full support Opera, iPhone, Android, Symbian, Samsung, Iris, Bolt, Firefox, MicroB.
Not supported Skyfire, IE, BlackBerry, NetFront, Obigo.
Note What I test here is whether the browsers take their data from the correct property pairs. Whether these property pairs give correct information is not part of this particular test.

pageX/Y is still relative to the page in CSS pixels, and this is by far the most useful of the three property pairs, just as it is on desktop.
