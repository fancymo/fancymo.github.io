AMD 规范

AMD规范是对 CommonJS 规范的延伸，它的模块定义如下：

AMD 需要在声明模块时指定所有依赖。

AMD 中需要 define 明确定义一个模块，在 Node 中是隐式包装的，目的是进行作用域隔离，仅在被需要的时候被引入，避免全局变量或者全局命名空间的方式，以避免变量污染和不小心被修改。

CMD 规范

CMD 与 AMD 区别主要在于定义模块和依赖引入的部分。

require、 exports 和 module 通过行参传递给模块，在需要依赖模块时，随时调用 require 引用即可。
