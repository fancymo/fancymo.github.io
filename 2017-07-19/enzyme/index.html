React 官方测试工具库

简单介绍下官方测试工具库，具体内容感兴趣可以去官网查看。

react-test-renderer/shallow react-dom/test-utils

Shallow Rendering

shallow rendering 指的是将一个组件渲染成虚拟的 DOM 对象，只渲染第一层（one level deep），不渲染子组件。

shallowRenderer.render()

把 shallowRenderer 理解为一个渲染待测试组件的空间（place），可以从中提取组件的输出。

shallowRenderer.render() 类似于 ReactDOM.render()，但它不需要 DOM 环境且仅渲染 a single level deep。

shallowRenderer.getRenderOutput()

返回的就是一个浅渲染的虚拟DOM对象。

测试工具

Simulate

模拟一个事件派发到 DOM 节点上，可以传入 eventData;

renderIntoDocument

调用 renderIntoDocument，将组件渲染成真实的 DOM 节点。

enzyme

官方测试工具库的封装，模拟 jQuery 的 API。enzyme 的设计是让不同底层处理引擎都具有相同的 API。

提供三种测试方式 shallow render mount。

概念

predicate 返回 true 或 false 的方法。

selector 类似于 CSS 选择器，也可以是一些在 enzyme 中的节点。

  1. 一个有效的 CSS selector

    • class syntax(.foo)
    • tag syntax(input, div, span)
    • id syntax(#foo)
    • prop syntax([bar], [bar=1]), Prop 选择器可以是string number 字面量、布尔值语法相结合的字符串语法

  2. React Component Constructor

  3. React Component’s displayName

  4. Object Property Selector

wrapper enzyme wrapper class,提供了一些 API。

可以通过节点属性的子集寻找组件。

Shallow Rendering (shallow)

shallow 方法是官方的 shallow rendering 的封装。确保测试不会间接维护子节点的行为。

shallow(node[, options]) => ShallowWrapper

  1. node: 需要渲染的节点
  2. options: context 传入组件的上下文，disableLifecycleMethods 如果设置为 true，componentWillMount 将不会被调用，且 setProps 和 setContext 后将不会调用 componentDidUpdate。

强制刷新

Full Rendering (mount)

mount(node[, options]) => ReactWrapper

  1. node: 需要渲染的节点
  2. options: context 传入组件的上下文，attachTo 组件要附加的 DOM 元素，childContextTypes 为所有的 children 合并 contextTypes。

提供的 API 与 shallow 类似

主要罗列独有 api 如下

根据组件的 ref 属性获取匹配元素
.ref(refName) => ReactWrapper

将 react tree 从 DOM 中卸载。(ReactDOM.unmountComponentAtNode()
.detach() => void

不支持 dive 等方法。

Static Rendering (render)

将 react 组件渲染为静态 HTML，分析 HTML 代码的结构。与 shallow、mount 主要不同是采用第三方 HTML 解析库 Cheerio，返回的是 Cheerio 实例对象。
