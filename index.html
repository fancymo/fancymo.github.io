React 官方测试工具库

Shallow Rendering

shallow rendering 指的是将一个组件渲染成虚拟的 DOM 对象，只渲染第一层（one level deep），不渲染子组件。

shallowRenderer.render()

把 shallowRenderer 理解为一个渲染待测试组件的空间（place），可以从中提取组件的输出。

shallowRenderer.render() 类似于 ReactDOM.render()，但它不需要 DOM 环境且仅渲染 a single level deep。

shallowRenderer.getRenderOutput()

返回的就是一个浅渲染的虚拟DOM对象。

Shallow Rendering (shallow)

shallow 方法是官方的 shallow rendering 的封装。确保测试不会间接维护子节点的行为。

shallow(node[, options]) => ShallowWrapper

  1. node: 需要渲染的节点
  2. options: context 传入组件的上下文，disableLifecycleMethods 如果设置为 true，componentWillMount 将不会被调用，且 setProps 和 setContext 后将不会调用 componentDidUpdate。

diff 是 Unix 很重要的工具程序。

  • 正常格式(normal diff)

上下文格式的 diff

二十世纪八十年代初，加州大学伯克利分校推出 BSD 版本的 Unix 时，觉得 diff 的显示结果太过简单，加入了上下文。

使用方法是加入 c 参数。

第一部分的两行, *** 表示变动前的文件，--- 表示变动后的文件。15个 *将文件的基本情况与变动内容分割开。

文件内容每一行最前面，还有一个标记位，如果为空该行无变化，如果为感叹号（!），表示该行有改动；如果为减号（-），表示该行被删除，如果是加号（+），表示该行为新增。

第四部分显示为变动后的文件，即 m2;

合并格式的 diff

假使两个文件的相似度很高，上下文格式的 diff 讲显示大量重复的内容。1990年，GNU diff 推出‘合并格式’的 diff。

第一部分的两行, --- 表示变动前的文件，+++ 表示变动后的文件。

第二部分，变动的位置用两个 @ 作为起首和结束。减号(-) 表示第一个文件，1 表示第1行, 7 表示连续7行。+1,3 表示变动后的。

与性能相关的那些事

发表于 2017-03-04

Web Storage

localStorage 是一个同步API，数据是存储在磁盘文件上，浏览器先将该文件读取到内存中，才能用 JavaScript 访问这些数据。

tips: 最佳策略是尽可能少使用键值对，存储尽可能多的数据。

关于内联

减少了 HTTP 请求次数，但损失许多其它方面的指标。

  1. 缓存损失
    HTML 文件承载所有资源，但 HTML 是不可以缓存。

  2. 没有边缘缓存
    在 CDN 端缓存静态资源。HTML 不能缓存，整个 HTTP 请求就没有被缓存，加载时每次都是要从源服务端重新获取，而不是从 CDN 访问，而标准情况下，部分资源是可以从边缘缓存获取。

  3. 没有按需加载
    资源被引用，条件满足时才会被下载。浏览器内置按需加载 CSS 图像的机制。

  4. 浏览器预加载失效
    浏览器使用只能启发模式提前获取页面底端的资源，执行前便 解析 DNS，并在执行前下载。

tips: HTTP请求头和响应头在 1kb 左右，将比它小的文件内联；页面中的图片尽量不要内联；与首屏无关紧要的内容不应该内联。
